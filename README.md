# 🚀延迟启动工具_管理Windows开机启动项_DelayedStartupToolPro
## 🎉公开源码，有缘人继续优化
不知道你有没有这种经历：按下开机键，桌面出来了，但接下来一两分钟里鼠标转圈、点啥都慢半拍——罪魁祸首往往不是电脑老了，而是**几十个程序在开机瞬间同时抢着启动**。
系统自带的启动项管理要么藏得深，要么只能“开/关”二选一，没法精细控制“谁先跑、谁晚点跑”。**Delayed Startup Tool Pro（延迟启动工具 Pro）** 就是来解决这个问题的。

#### 它到底是什么？
一款轻量的 Windows 启动项管理工具，核心能力只有一句话：**给每个开机自启动程序单独设置“延迟多少秒再启动”**，让它们错峰运行，开机瞬间不再挤成一团。
界面采用 WPF + WebView2 打造，深色主题、卡片式布局，操作逻辑和主流效率软件一致，上手零门槛。

#### 核心功能
**1. 可视化延迟调度**
每个启动项都有独立的延迟滑块（秒级），也支持直接点数字手动输入精确秒数。状态一目了然：立即启动 / 延迟启动 / 已禁用。

**2. 一键智能优化**
不想一个个调？点“一键优化”，工具会根据程序类型（系统目录、知名软件签名、普通程序）自动分配最合理的延迟梯度，把原本“全员齐发”变成“有序进场”，显著降低开机卡顿。优化前会自动快照，随时可“恢复默认”一键回退。

**3. 系统启动项扫描**
一键扫描注册表 `Run` 键、启动文件夹、计划任务三大来源里的所有自启动程序，并标出正在运行的任务，帮你发现那些“偷偷开机就跑”的陌生程序。

**4. 三种启动模式**
支持 GUI（普通窗口）、BAT（批处理）、后台隐藏 三种启动方式，适配不同类型的程序。

**5. 开机自启动与拖拽排序**
可以将工具自身或任意程序“添加到开机启动”，列表支持拖拽排序决定先后；右键菜单还提供“立即启动”“启用/禁用”“编辑路径/参数/备注”等快捷操作。

**6. 健壮与稳定**
内置三层全局异常兜底 + 全局错误提示，长时间运行不易崩；内存占用做了优化，常驻也轻量。

#### 适合谁用？
- 开机后习惯性卡顿、想根治“启动风暴”的普通用户
- 装机党、系统优化爱好者、工程人员
- 需要在多台电脑上统一启动策略的进阶用户

#### 一句话总结
不是让你删程序，而是让它们**排队启动**——开机快一点，电脑稳一点。

## 系统要求
操作系统：Windows 10 / Windows 11（仅 x64 架构；ARM 设备可经 x64 仿真运行，源码自行修改支持X86以及Win7）

最低权限：需管理员权限：程序清单已声明 requireAdministrator，每次启动都会弹 UAC 提权、以管理员身份运行；修改计划任务 / 系统级启动项 / 注册表 Run 键等敏感操作天然具备权限，无需额外右键提权

### 关于 SmartScreen / 杀软拦截

- 当前程序使用**自签名证书**（作者自签），**不受 Windows SmartScreen 信任**。
- 表现：首次运行 / 下载后可能被 SmartScreen 拦一道，提示“Windows 已保护你的电脑”或“未知发布者”。
- 解决：点击“**仍要运行**” / “更多信息 → 仍要运行”即可，不影响功能。
- 想彻底消除拦截：需用正规 CA 颁发的代码签名证书（.pfx）重签——把证书路径和密码设为环境变量 `SIGN_PFX` / `SIGN_PWD` 后签名。

### 常见问题 FAQ

**Q：打不开，提示缺少 .NET 运行库？**
A：你用的是“框架依赖版”，请到 <https://dotnet.microsoft.com/download/dotnet/8.0> 装 “Windows Desktop Runtime 8.0.x (x64)”；或改用“自包含版”（自带运行库，体积大但免装）。

**Q：界面是空白的 / 一直转圈？**
A：界面基于 WebView2。本程序已打包 WebView2 运行时，正常离线可跑；若仍空白，请确认解压完整（别只拷了 exe，要把整个目录一起拷），并以管理员身份运行（本程序启动即提权，若被拦请在 UAC 中点"是"）。

**Q：修改启动项没反应 / 提示权限不足？**
A：本程序默认就以管理员身份运行（启动即 UAC 提权）。若仍提示权限不足，通常是目标键值被组策略 / 其他软件锁定，或 UAC 启动时点了"否"——请重新以管理员身份运行本程序并在 UAC 中点"是"；若开着 UAC 且频繁异常，可在程序提示时选择关闭 UAC 后重试。

**Q：一键优化后想反悔？**
A：点“恢复默认”即可——优化前已自动快照，一键还原。

**Q：支持 ARM 笔记本吗？**
A：当前只发 x64 构建。ARM 设备（如 Surface 骁龙版）可经系统 x64 仿真运行，但未经专门适配，建议优先在 x64 机器使用。
